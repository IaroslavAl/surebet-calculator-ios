# START HERE

Документ для быстрого старта агента: что читать, как действовать, что проверять.

## Когда открывать
- Любая новая задача в проекте.
- Любая задача, где неочевидно место изменений.

## Быстрый triage (60-120 секунд)
1. Определи тип задачи:
   - изменение UI/ViewModel;
   - изменение сервиса/сети;
   - изменение архитектуры/модуля;
   - добавление/починка тестов;
   - изменение CI/pipeline;
   - документация.
2. Открой `docs/agent/POLICIES.md` и проверь ограничения.
3. Открой релевантный playbook из `docs/agent/PLAYBOOKS/`.
4. Уточни область кода в `docs/reference/PROJECT_MAP.md` и `docs/reference/MODULE_INDEX.md`.

## Карта принятия решений
- Нужно понять где код: `docs/reference/PROJECT_MAP.md`.
- Нужно понять кто от кого зависит: `docs/reference/DEPENDENCY_GRAPH.md`.
- Нужно понять контракты модулей: `docs/reference/MODULE_INDEX.md`.
- Нужно понять команды проверки: `docs/reference/BUILD_TEST_COMMANDS.md`.
- Нужно понять CI-контракт и required checks: `docs/reference/CI_RULES.md`.
- Нужно избежать повторения старой ошибки: `docs/lessons/INCIDENTS.md`.

## Definition of Done для агента
- Выполнены релевантные MUST-правила из `POLICIES.md`.
- Изменения в коде согласованы с архитектурой и контрактами модулей.
- Выполнены обязательные команды валидации (build/test по ситуации).
- Документация обновлена, если изменилась архитектура/контракты/правила.

## Анти-паттерны
- Изменять ViewModel обходя `send(_:)`.
- Хардкодить UI-строки.
- Протаскивать `Binding` во ViewModel как внешний mutable state.
- Дублировать один и тот же факт в 3+ документах без источника правды.
