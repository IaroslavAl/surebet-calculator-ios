# PLAYBOOK: CHANGE NETWORK SERVICE

## Когда использовать
- Изменения в сетевых сервисах и кэше (например, `Banner/Service.swift`).

## Шаги
1. Проверить контракты модели/декодирования.
2. Убедиться в потокобезопасности mutable state.
3. Обновить ошибки и fallback-поведение.
4. Добавить/обновить unit-тесты c `MockURLProtocol`.
5. Для test URL использовать уникальные полные URL.
6. Запустить `build` + `test`.

## Чеклист PR
- Нет глобальных race-prone handler-ов в тестах.
- Кэш-консистентность сохраняется при ошибках.
